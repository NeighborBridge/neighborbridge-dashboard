NeighborBridge 沙盒合规性

轻量级合规工作面板 — 明确 FASD/NDD 教育产品、CHW workflow demo 和 FQHC/MCO partner discussion 的安全边界

📌 1. 沙盒定义

Sandbox = low-risk, non-PHI, non-clinical-decision testing environment for education and workflow demonstration only.

NeighborBridge sandbox 当前定位为：非 PHI、非临床决策、非真实患者管理系统的教育与流程测试环境。它可以用于展示能力、验证工作流、生成教育材料和模拟路径，但不能用于处理真实患者资料或提供临床决策。

✅ 用于

  • Education
  • Workflow demonstration
  • Implementation planning
  • Partner discussion

❌ 不用于

  • Diagnosis
  • Treatment
  • Clinical decision-making
  • Storage or processing of PHI

🔒 2. PHI 边界

✅ 允许 — 仅虚构/合成数据

  • 虚构案例研究
  • 合成数据 / 去标识化教育场景
  • 通用 CHW 培训材料
  • 模拟转诊路径

❌ 不使用

  • 真实患者姓名、出生日期、病历号、地址
  • 真实照片、录音、病历截图
  • 真实临床笔记
  • 真实护理计划

✅ 3. 允许的内容

  • ✓Public education materials
  • ✓Synthetic family scenarios
  • ✓De-identified workflow examples
  • ✓Generic CHW scripts
  • ✓Mock referral pathways
  • ✓Sample dashboard screenshots with fake data
  • ✓AI 生成的合成案例
  • ✓非 PHI 的 CHW 教育脚本
  • ✓模拟家庭导航流程
  • ✓模拟 FQHC / MCO 演示材料
  • ✓Dashboard 文案草稿
  • ✓Partner-facing 非机密说明材料
  • ✓Agent workflow 测试任务

❌ 4. 禁止的内容

  • ✗Identifiable patient data
  • ✗Real clinic schedules
  • ✗Real messages from families
  • ✗Actual lab / imaging / diagnostic reports
  • ✗Provider notes copied from EHR
  • ✗Any PHI used in AI tools or dashboards
  • ✗真实患者病历内容
  • ✗真实 EHR note
  • ✗家属真实短信、录音、照片
  • ✗可识别的 care plan
  • ✗真实 clinical decision support
  • ✗任何需要 HIPAA / BAA 审查后才能处理的资料
  • ✗把 sandbox 当成真实 pilot 或 live clinical system 使用

⚠️ 5. 演示免责声明

"This sandbox is for education, workflow demonstration, and implementation planning only. It does not contain PHI and is not used for diagnosis, treatment, or clinical decision-making."

"本 sandbox 仅用于教育、流程演示和实施规划，不包含受保护健康信息，不用于诊断、治疗或临床决策。"

🤝 6. 合作伙伴展示策略

  • 对 Trillium / FQHC / academic partners 展示时，只展示模拟路径、合成案例和通用教育流程
  • 真实实施前必须由 partner organization 完成：
    • HIPAA 合规审查
    • BAA (Business Associate Agreement) 签署
    • IT security 评估
    • Clinical governance 审查
  • 任何时候不暗示 sandbox 内容代表真实临床数据或已获机构批准

🤖 9. AI 工具选择 / AI Tool Strategy

当前阶段最适合：

  • ChatGPT Team / Enterprise 或 OpenAI API：用于非 PHI 教育材料、合成案例、CHW 脚本、dashboard 文案和 partner-facing 草稿。
  • OpenClaw + DeepSeek：适合做技术 sandbox / agent workflow sandbox，例如 dashboard 自动更新、任务分配、synthetic case 测试、非 PHI workflow demo。
  • GitHub + Cloudflare dashboard：适合展示 synthetic workflow 和 partner-facing demo 页面。

未来真实 pilot 阶段：

  • 若进入 FQHC / Trillium / MCO 真实试点，应由 partner 的 IT / compliance / legal 团队决定使用 Azure OpenAI、AWS Bedrock、Google Vertex AI 或其他可签 BAA、可通过 HIPAA/security review 的企业环境。
  • 在真实 pilot 前，NeighborBridge 不应自行把 PHI 放入任何外部 AI 工具。

🛠️ 10. OpenClaw + DeepSeek 使用边界

OpenClaw + DeepSeek 可以继续作为 NeighborBridge 的 non-PHI technical sandbox 使用，但必须保持明确边界：

✅ 允许

  • 非 PHI workflow demo
  • Dashboard 自动更新
  • Synthetic case 测试
  • CHW script 草稿
  • Agent task routing
  • 内部 operational prototype

❌ 禁止

  • 真实患者资料
  • EHR notes
  • 家属真实短信/录音/照片
  • Clinical decision support
  • Partner-facing HIPAA pilot
  • 任何可识别患者或家庭的信息

"OpenClaw / DeepSeek sandbox is for non-PHI technical workflow testing only. No real patient data, no EHR text, no identifiable family communication, and no clinical decision support."

"OpenClaw / DeepSeek 沙盒仅用于非 PHI 技术流程测试，不得输入真实患者数据、EHR 文本、可识别家庭沟通内容，也不得用于临床决策支持。"
